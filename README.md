# 캡스톤 졸업작품: 블록체인 기반 의료장비 관리 시스템

> 의료장비의 **등록-점검-수리-폐기** 전 주기를 위변조 방지 가능한 이력으로 관리하기 위한 프로젝트

## 프로젝트 한 줄 소개
병원 내 의료장비 관리 이력을 블록체인에 기록해 데이터 신뢰성을 높이고, 웹 기반 관리 화면으로 운영 효율을 개선하는 캡스톤 졸업작품입니다.

## 문제 정의
기존 의료장비 관리 방식은 다음과 같은 한계가 있습니다.
- 엑셀/수기 기반 관리로 인해 이력 누락 가능성 존재
- 누가 언제 어떤 변경을 했는지 추적 어려움
- 장비 점검/수리 기록의 무결성 검증 어려움

## 해결 방식
본 프로젝트는 **오프체인 + 온체인 하이브리드 구조**를 적용합니다.
- 조회 성능이 필요한 데이터(장비 목록/검색)는 DB(오프체인)에 저장
- 신뢰성이 필요한 이벤트(등록/점검완료/수리완료/상태변경)는 블록체인(온체인)에 해시/트랜잭션으로 기록
- 관리자·엔지니어·의료진 역할 기반 접근 제어로 업무 범위를 분리

## 핵심 기능
1. **장비 등록/조회**
   - 장비번호, 모델명, 제조사, 설치 위치, 도입일, 상태 관리
2. **점검 이력 관리**
   - 정기 점검 일정, 점검 결과, 점검자 기록
3. **수리/교체 이력 관리**
   - 고장 접수, 수리 내용, 교체 부품, 처리 일시 기록
4. **상태 대시보드**
   - 정상/점검필요/수리중/폐기예정 상태를 시각적으로 확인
5. **감사 추적(Audit Trail)**
   - 변경 주체, 변경 시점, 변경 내용을 추적
6. **블록체인 검증**
   - 주요 이력의 트랜잭션 식별자(해시)로 위변조 여부 확인

## 기대 효과
- 의료장비 이력의 위변조 방지 및 책임 추적성 강화
- 점검/수리 누락 감소를 통한 환자 안전성 향상
- 인증/감사 대응 시 신뢰 가능한 증빙 데이터 확보
- 장비 운영 데이터 축적으로 유지보수 최적화 기반 마련

## 시스템 구성도
```text
[사용자 웹 클라이언트]
          |
          v
      [백엔드 API]
      |         |
      |         +--> [운영 DB (오프체인 데이터)]
      |
      +------------> [블록체인 네트워크 (이력 검증 데이터)]
```

## 역할 정의
- **관리자**: 장비 등록/수정, 사용자 권한 관리, 전체 이력 조회
- **엔지니어**: 점검/수리 이력 등록 및 처리
- **의료진(조회권한)**: 장비 상태 및 이력 조회

## 데모 시나리오 (발표용)
1. 관리자 로그인 후 신규 장비 등록
2. 엔지니어가 정기 점검 결과 입력
3. 고장 이벤트 생성 후 수리 내역 등록
4. 장비 상태가 `수리중 -> 정상`으로 변경됨을 확인
5. 해당 변경 이력의 블록체인 트랜잭션 정보 확인

## 저장소 상태 안내
현재 저장소는 초기 단계이므로, 실제 실행 명령/세부 기술 스택은 구현 코드 추가 후 아래 항목을 업데이트하면 됩니다.
- 실행 방법
- 환경 변수 목록
- 배포 방법

## 향후 고도화 계획
- IoT 센서 연계로 장비 상태 자동 수집
- 이상탐지 기반 고장 예측 알림
- 병원 간 이력 연동을 위한 표준 데이터 모델 설계
- 전자서명 기반 승인 프로세스 도입
